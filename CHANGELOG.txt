CHANGELOG — Tropical Roots Maui - Plant Manager
=================================================

v2.25.1 — 2026-07-18
--------------------
FIX: VPD Trends chart drew a flat line

  The VPD Trends chart on the Analytics tab read the VPD calculator's result
  as an object (r.vpd) when it returns a plain number, so every point plotted
  as zero and the line lay flat on the axis. It now reads the value directly
  and plots real VPD from your environment readings. Affected every user with
  environment data, not just the demo.

POLISH: Analytics default range now 90 days

  The default analytics time window was 30 days, too short for month-long grow
  cycles, so yield-over-time and cost trends often showed too little to be
  useful. Default is now 90 days (still adjustable per chart view).

FIX: Demo data completeness

  Several Analytics charts showed empty states in the demo garden because the
  demo fixture wrote values the app doesn't use. Corrected so the demo now
  fully populates:
  - Feeding EC fields (ecIn/ecOut -> ecPpmIn/ecPpmOut) for Feeding EC/PPM
    Drift and Feeding -> Yield
  - Per-plant dry-back pot config for DryBack Curves
  - IPM severity (mild/moderate -> low/medium) for IPM Issue Overview
  - Harvest cure status (cured -> complete) and seed type
    (Regular/Auto -> regular/autoflower)
  - Environment and water temperatures stored in Fahrenheit (the app's unit)
    so VPD and temperature trends compute correctly

v2.25.0 — 2026-07-17
--------------------
FEATURE: Hawaiian Heritage Landraces

  The strain catalog gains a Hawaiian Heritage Landraces block: Hawaiian
  Sativa, Kaua'i Electric, Puna Buddaz, Maui Wowie (Landrace), Hawai'i '78,
  Moloka'i Purpz, and The Blood, with type, origin lore, and flavors where
  documented.

  The demo garden now grows this lineup end to end. All six demo mothers
  (Maui Wowie through The Blood), the strain library, the seed bank, the
  pheno hunt (now the Hawai'i '78 Seed Run), the breeding crosses (Wowie x
  Purpz F1), terpene profiles, and the activity log are island-grown.

FEATURE: Storage Protection

  On load the app asks the browser to protect its stored data from eviction
  (navigator.storage.persist). Installed PWAs on Chromium get this granted
  automatically; on iOS it reduces the chance Safari clears local data under
  disk pressure. About shows "storage protected" when granted. Backups are
  still recommended; this is a shield, not a substitute.

FIX: Plant-card More menu readability

  The overflow menu on plant cards used a translucent background, so cards
  behind it bled through and made menu items hard to read. It now uses an
  opaque surface, sits above neighboring cards, and flips upward when there
  is no room below so it never runs off-screen.

POLISH: Tips & counts

  Dashboard tips corrected: the Analytics tip now says over thirty charts
  (was fourteen), the strain-catalog tip reflects 370+ entries and the new
  landraces, and the feed-schedule tip includes Down To Earth.

v2.24.0 — 2026-07-14
--------------------
FIX: iOS reload loop

  Fixed a bug where the app could continuously reload itself on iOS (home-
  screen / standalone PWA). The service worker auto-activated every update and
  the page reloaded whenever a new worker took over — a chain that iOS re-
  triggered on each launch, producing a nonstop reload. Updates are now user-
  prompted: a new version installs quietly in the background and a "new
  version available" notice lets you reload when you choose, so you never lose
  what you were typing. Thanks to more_greens for reporting this.

FEATURE: Plant Cost Breakdown & Archive Reasons

  The Strain Comparison tool now breaks cost down by plant outcome. Every
  expense is treated as direct (tagged to a specific plant) or shared
  overhead (untagged); overhead is split equally across every plant grown.
  Per strain you now see Cost (harvested plants), Lost (dead plants), Cost
  per Plant, and a corrected Cost per Gram (harvested cost over yield).

  The old Cost/Gram was miscomputed — it used total garden expense times
  average harvest weight, which is a dollar figure, not a per-gram rate.
  It is now a proper per-strain value.

  Archiving a plant now prompts for a reason — Harvested, Retired, Died,
  or Culled — stored as archiveReason. Died and Culled plants drive the
  new "money lost" figure. Plants archived before this release (or via
  batch archive) have no reason and are not counted as losses.

  Documented in the Help Guide FAQ ("How are the plant costs in Strain
  Comparison calculated?").

FEATURE: Nutrient Mix Calculator — Dry Amendments (tbsp/gal)

  Each product row now has a unit selector (ml/gal or tbsp/gal), so dry
  amendments (Epsom salt, cal-mag powder, top-dress) can be mixed
  alongside liquid nutrients. Results and saved presets carry the per-row
  unit; older presets load as ml/gal.

FEATURE: Medium Fill Calculator

  New calculator that works out how much growing medium fills a bed, pot,
  or tent footprint. Enter rectangular (bed/tent) or round (pot)
  dimensions in feet, inches, or cm, with quantity and fill-level inputs.
  Outputs total volume in cubic feet, gallons, and liters, plus how many
  bags you need at 1.0 / 1.5 / 2.0 / 3.0 cu ft each. A "Build this mix in
  the Soil Mix Calculator" button hands the volume straight over to the
  Soil Mix Calculator. Available from the Dashboard, gear menu, and
  Command Palette.

FEATURE: Shareable Tool Links

  Most of the app's tools (31) now have a Share button that hands out a deep
  link (e.g. .../?tool=soilmix). Opening that URL jumps straight into the
  tool. Uses the native share sheet on mobile and the clipboard on desktop;
  the params are stripped after they fire so a reload doesn't re-open it.

  Covered: every calculator (Soil Mix, Medium Fill, VPD, DLI, Nutrient Mix,
  KNF, CO2, pH Chart, Light Map / PPFD, Power Cost, Unit Converter); the
  growing tools (Strain Comparison, Plant Comparison, Harvest Forecast, Grow
  Cycles, Grow Runs, Pre-Grow Checklist, Garden Stats, Terpene Profiles, Crop
  Steering, Environmental Recipes, Water Quality Log, Spray Rotation,
  Breeding Tracker, Pheno Hunt, Ferment Tracker, Seed Bank); and the
  diagnostic tools (Deficiency Wizard, Pest Identifier, Visual Pest Guide,
  Companion Planting Guide).

  For VPD, DLI, Medium Fill, and KNF the link also carries your input values,
  so you can share an actual worked calculation — e.g. .../?tool=vpd&t=82&rh=45
  opens the VPD calc already showing that result, and
  .../?tool=knf&vol=5&unit=gal&mode=drench opens the KNF dilutions for a
  5-gallon soil drench. (Soil Mix and Nutrient Mix open without pre-fill;
  the diagnostic tools are wizards / reference, so they open to the start.)

POLISH: Calculator Labels & Soil Recipe

  - Soil Mix Calculator: Base Mix and Aeration sections now labeled
    "parts by volume" to match the Amendments header
  - Soil Mix Calculator: added cubic feet (cu ft) as a volume unit for
    growers who buy soil by the bag; base and aeration amounts now display
    in the unit you select (previously always gallons)
  - Soil Mix Calculator: Biochar added as a selectable base component
    (high-CEC carbon; charge before use)
  - Tropical Living Soil preset refined for the island climate: coco-based
    rather than imported peat (better in a hot, wet climate and more
    sustainable to land on Maui), with Biochar for high CEC (the
    nutrient/water-holding peat would give, without the downsides), plus
    dolomite for coco's Ca/Mg, gypsum, and slow-release rock phosphate
  - DLI Calculator: selecting Sunlight now clears the wattage field (a
    fixed-PPFD source) instead of leaving a stale value
  - Added missing units: CropSteer "EC Runoff (mS/cm)", DLI "Wattage (W)"
  - Feed Schedules: added Down To Earth Organic as a 5th built-in template
    (All Purpose 4-6-2 -> Bud & Bloom 0-10-10 dry-amendment top-dress program)
  - Feed Schedules: the organic schedules (Gaia Green, Down To Earth) now
    finish on "water only" instead of a pre-harvest flush — flushing only
    applies to mineral/salt nutrient lines (GH Flora, Fox Farm)

POLISH: Demo Data Coverage

  Filled out the demo fixture so every tool has sample data to explore:
  a pheno hunt (an active Wedding Cake seed run with four scored
  phenotypes), three terpene profiles (OG Kush, Granddaddy Purple, Blue
  Dream) on cured harvests, a PPFD light map, a saved soil recipe, and
  four KNF ferments (FPJ, LABS, FAA, WCA). Because ferments are stored in
  settings rather than a top-level collection, Clear Demo Data now
  surgically removes demo ferments while leaving any you started yourself
  untouched.


v2.23.4 — 2026-06-09
--------------------
SECURITY: QR Decoder & Export Hardening

  The QR Lookup decoder now escapes the numeric-looking fields (seed
  counts, germination %, cutting and success counts) parsed from a
  scanned or pasted tag, closing the last unescaped output path in the
  decoder. A crafted QR string can no longer inject markup on lookup.

  Plaintext backup files now strip user-supplied secrets — the weather
  and Google Vision API keys and the Firebase config — so a shared
  backup cannot leak credentials. Encrypted backups retain them, since
  the AES-256-GCM password already protects the file; restoring an
  encrypted backup brings your keys back automatically.

POLISH: Attribute-Escaping Consistency

  Wrapped the remaining unescaped data-* attribute values (nutrient
  product default amount, IPM treatment category) in the now quote-safe
  escaper for defense-in-depth.

SEO: Structured-Data Upgrade

  Upgraded the homepage JSON-LD from WebApplication to
  SoftwareApplication with featureList, screenshot, and softwareVersion,
  and added a BreadcrumbList to the Help Guide.


v2.23.3 — 2026-06-09
--------------------
SECURITY: Output-Encoding Hardening (XSS Defense-in-Depth)

  Hardened the central HTML-escaping helper so it also encodes double
  and single quotes. This closes attribute-context escape vectors where
  user- or import-supplied text (strain names, breeders, nicknames,
  notes, environment names, etc.) is rendered into HTML attribute
  values. Element-text rendering is unchanged.

  Also wrapped every stored-photo image source in the escaper, so a
  crafted photo field inside an imported backup can no longer break out
  of the src attribute. In-app photos (canvas re-encoded data URLs) are
  unaffected.

  Maintenance release: no feature or data-format changes; existing data
  and backups render identically.


v2.23.2 — 2026-05-11
--------------------
FEATURE: Web Share Target — Receive Shared Photos from Android

  The manifest now declares a share_target endpoint so Plant Manager
  appears in the Android system share sheet for any image. Snap a photo
  in the Camera app, hit Share, pick Plant Manager, and the photo lands
  pre-attached in the Add Journal Photo modal — ready for plant
  selection, growth-stage tagging, and notes. The service worker
  intercepts the multipart POST, stashes the file in a dedicated
  trm-share-target cache under /_share/photo (with the original
  filename in an X-Filename header), then 303-redirects to a GET so
  the page can load and consume the stash on init.
    - New Journal.openAddModalWithFile(file) external entry point
      reused by both share-target and the existing flow
    - The trm-share-target cache survives the activate-purge so a
      freshly-shared photo isn't deleted before consumption
    - Cache entries are deleted after consumption so a normal reload
      doesn't re-fire the import

FEATURE: Tab-Nav Badge Counts

  Numeric pill badges now appear on tab-group buttons when there's
  something needing attention — visible on every tab, no need to open
  each one to check:
    - Care: count of overdue + due-today tasks (red, amber if >5)
    - Plants: count of unresolved IPM issues (red)
    - Insights: count of environment alerts vs. configured thresholds
      (amber)
  Badges hide when count is zero. Updates on every Store change via
  the existing Store.subscribe loop. Display "99+" when count exceeds
  99 to avoid layout shift.

FEATURE: Sticky Toolbar Headers on Long Tabs

  The .toolbar bar (Add buttons + filter selects) is now position:sticky
  with top calculated from the fixed header + tab nav heights. Filter
  re-selection and quick-add buttons stay reachable while scrolling
  through 200+ photos / 50+ plants. Subtle bottom border and
  background match the panel; print stylesheet reverts to static so
  reports aren't affected.

FEATURE: Drag-and-Drop + Clipboard Paste for Photo Uploads

  The Add Journal Photo modal preview tile now accepts:
    - Drag-and-drop from any source (Files app, Photos, browser tabs)
      with cyan dashed outline feedback during drag-over
    - Clipboard paste via Ctrl+V — perfect for screenshots taken with
      Win+Shift+S / Cmd+Shift+4
  Both paths flow through the same Journal._processFile(file) pipeline
  as the file picker, so EXIF date extraction, orientation correction,
  and resize all apply identically. Paste listener auto-cleans-up
  when the modal DOM goes away.

FEATURE: EXIF Orientation Correction

  Phone-camera portraits no longer end up sideways after upload.
  New EXIF.readOrientation(file, cb) reads the JPEG's Orientation
  tag (0x0112). U.resizeImage now honors orientations 1-8, swapping
  width/height for orientations 5-8 and applying the appropriate
  canvas transform (translate/scale/rotate per the standard EXIF
  matrix) before re-encoding. Especially relevant for Android
  cameras that store landscape pixel data with an Orientation=6
  rotation hint.

POLISH: SW Update Notification — User-Controlled Refresh

  When a new service worker is installed and waiting, the page no
  longer silently force-reloads (which was eating mid-form work).
  Instead a sticky "A new version is available" toast appears with a
  Reload action button. The user clicks Reload when ready; the SW
  receives SKIP_WAITING and the existing controllerchange listener
  performs the reload after the new SW takes over.
    - Toast.show now supports duration:0 for sticky toasts that
      only dismiss on click
    - Catches three cases: page loaded with reg.waiting already
      present, updatefound while page is open, and post-skipWaiting
      controllerchange

POLISH: Smart Install Prompt Gating

  InstallPrompt._showOneTimeNudge no longer fires the install toast
  the first time Chrome surfaces beforeinstallprompt — that's often
  before the user has any reason to want the app installed. New
  gating: defer until the user has 3+ plants OR 5+ feedings
  (engagement signals that say "this person is invested"). Subscribes
  to Store changes to re-evaluate after every save. Falls back to a
  90-second session-time timer so a casual user still sees the
  offer eventually.

POLISH: Service Worker — Single Fetch Listener

  Consolidated the share-target POST handler and the existing fetch
  strategy into one fetch event listener (avoids double-respondWith
  errors). Cache version bumped to 2.23.2.


v2.23.1 — 2026-05-11
--------------------
FEATURE: Web Share API — Native Share from Snapshot + Reports

  navigator.share is now wired into three places that previously
  forced a "Copy to clipboard then paste somewhere else" workflow:
    - Daily Snapshot modal — new "Share..." button (visible only
      on browsers that support navigator.share) shares the snapshot
      text via the OS share sheet
    - Plant Report / Garden Report popup — new Share button in the
      top-right that wraps the rendered HTML as a .html File and
      calls navigator.share with files; falls back to text-share
      if files aren't accepted; auto-hides on browsers without
      navigator.share
    - Grow Journal Export popup — same treatment as Plant Report
  Backed by a new Share helper module (window.Share) that wraps
  navigator.share with text() and files() helpers and graceful
  fallback (returns false instead of throwing on AbortError).

FEATURE: Manifest file_handlers + launch_handler

  The installed PWA now registers as a handler for application/json
  files. On Android (and desktop Chromium >= 102), opening a Plant
  Manager .json backup from the Files app launches the installed
  app with the file pre-loaded. The new launch_handler entry uses
  client_mode "navigate-existing" so a new file open reuses the
  running window instead of spawning a duplicate.
    - New ?action=import URL route consumes window.launchQueue
      (Chromium >= 102) and pipes the file straight into
      DataIO.handleImport(file) — no manual file-picker step
    - Older Chromium without launchQueue falls back to firing the
      import file picker so the user can re-pick

FEATURE: Offline Indicator Chip in Header

  Small amber pill that appears in the header when navigator.onLine
  is false. Includes inline SVG WiFi-off glyph. Tooltip explains
  that edits are still saved locally and will sync when reconnected.
  New Offline module listens to online/offline events and toasts a
  brief confirmation in both directions ("Offline — your edits are
  saved locally" / "Back online"). Suppresses the initial-load
  toast so users aren't pinged on every page load if they happen
  to be offline at boot.

FEATURE: Haptic Feedback on Mobile

  navigator.vibrate is now called on:
    - Toast Undo button press (10ms) — confirms the undo lands
      before the toast fades
    - Task completion (15ms) — physical "tick" on each completed
      task so the action feels deliberate
  No-op on desktop / when blocked by the OS; wrapped in try/catch
  so a missing vibrate API can't break the parent action.

POLISH: SW Network Timeout — No More Hangs on Captive Wi-Fi

  The network-first shell fetch in service-worker.js is now wrapped
  in a Promise.race against a 3-second NET_TIMEOUT_MS. Slow or
  captive networks no longer hang the page indefinitely; after the
  timeout the SW falls back to whatever's in the cache, then to
  '/' as a last resort. Also added /PlantManagerHelp.html,
  /plant-manager-vs-grow-with-jane.html, /privacy-policy.html, and
  /CHANGELOG.txt to PRECACHE so first offline visit to any of them
  works.

POLISH: Toast Sticky Mode

  Toast.show now treats opts.duration <= 0 as "stay visible until
  the user clicks". Used by the v2.23.2 SW update notification, but
  available for any toast that needs to wait for an explicit user
  decision rather than a time-based dismiss.

FIX: Cheat Sheet (?) Self-Toggle

  Pressing ? to close the keyboard cheat sheet was being eaten by
  the _modalOpen() guard, leaving the sheet stuck open until the
  user used Esc / overlay click. The ? handler now runs BEFORE the
  modal-open guard and toggles the sheet on/off via a
  Shortcuts._cheatSheetOpen flag that's auto-cleared by a
  MutationObserver on the modal overlay (so any close path resets
  it correctly).

FIX: Safari Download Cancel — Defer URL.revokeObjectURL

  The two object-URL revoke calls in DataIO (plain export +
  encrypted export) ran synchronously immediately after a.click(),
  which caused Safari to occasionally cancel the download mid-save.
  Both revokes now wrapped in setTimeout(..., 0) so the download
  finishes before the URL is invalidated.


v2.23.0 — 2026-05-10
--------------------
FEATURE: Install App (PWA prompt, properly captured)

  New InstallPrompt module captures the deferred beforeinstallprompt
  event on Android/Chrome browsers that fire it, and surfaces installation
  via three paths:
    - One-time toast nudge the first time the prompt is captured
      ("Install Plant Manager for offline use and a home-screen icon"
      with an Install action button) — gated by an installNudgeShown
      flag so it only ever fires once
    - Command Palette entry: Ctrl/Cmd-K → "Install App"
    - Gear menu Quick Actions: "Install App"
  Listens for the appinstalled event to clear the deferred prompt and
  toast a success message. Detects existing PWA installs via
  matchMedia('(display-mode: standalone)') and navigator.standalone so
  the install paths no-op when the app is already installed.

FEATURE: iOS Add-to-Home-Screen Hint

  Safari on iOS doesn't fire beforeinstallprompt, so the installable-PWA
  affordance has been invisible there. New one-time modal (~2.5s after
  init on first iOS launch) walks the user through the three-step
  Share → Add to Home Screen → Add flow with an inline SVG of the iOS
  share glyph so the user can pattern-match the button. Reachable on
  demand from the same Install App command/menu — calling
  InstallPrompt.prompt() on iOS opens this hint rather than failing
  silently. Gated by an iosInstallHintShown flag.

FEATURE: Auto Theme (follows system prefers-color-scheme)

  Theme module is now a tri-state: dark → light → auto → dark on each
  toggle tap. Auto resolves to the user's OS theme via
  matchMedia('(prefers-color-scheme: dark)') and subscribes to the
  media query's change event, so the app re-themes in real time when
  the OS flips at sunset/sunrise or via system shortcut.
    - Small "A" badge appears on the theme toggle button when Auto is
      active so users can tell which mode they're in
    - aria-label cycles through three descriptive strings ("Theme:
      Dark — tap for Light", etc.)
    - <meta name="color-scheme"> tag is now updated dynamically (set
      to "light dark" in auto mode) so form controls, scrollbars, and
      native widgets follow the chosen mode
    - Brief toast confirms when Auto mode is entered to avoid the
      "why did my theme just change?" puzzle on the first OS theme flip

FEATURE: About Dialog (tap the version chip)

  The footer version number is now a real button instead of a static
  link to /CHANGELOG.txt. Tapping it opens an About modal with:
    - Version + "installed PWA" marker when running standalone
    - "by Tropical Roots Maui" credit linking to the main site
    - Quick-action grid: What's New, Full Changelog, GitHub repo,
      Help Guide
    - GPL-3.0-or-later license footer
    - Explicit no-telemetry / no-tracker / no-ads statement
  Also reachable via Command Palette → "About Plant Manager".

FEATURE: Cloud Restore Shows the Diff Preview

  CloudSync.restore() previously asked a bare yes/no via Modal.confirm
  before destructively overwriting local data. It now wraps the
  Firestore payload in a synthetic _trm_export envelope and reuses
  DataIO._showImportPreview() — the same per-category record-count
  diff that file imports use. Users see:
    - Backup version + relative export time
    - Per-category Now / Backup / Δ table
    - "Backup has N fewer records than your current data" warning
  Before the "Restore from cloud" button is clickable. _showImportPreview
  was generalized with optional {title, intro, applyLabel} opts so file
  import and cloud restore share one render path with appropriately
  worded copy.

FEATURE: Run Setup Wizard, Replayable

  Setup Wizard is no longer one-shot. Two new entry points re-enable it
  at any time by clearing the setupWizardDismissed flag and resetting
  _step:
    - Command Palette: Ctrl/Cmd-K → "Run Setup Wizard"
    - Gear menu Quick Actions: "Run Setup Wizard"
  Toast confirms when the wizard is restored to the dashboard.

A11Y: Main Modal Gets Dialog Semantics + Focus Trap

  The main #modal element previously had no ARIA role or focus
  management — only the lightbox and command palette did. Now:
    - role="dialog", aria-modal="true", aria-labelledby="modalTitle"
    - Full Tab/Shift-Tab focus trap keeps focus inside the modal
      while it's open
    - On open, focus lands on the first input/select/textarea; if
      there isn't one, the first footer button (or the close button)
      is focused instead so Esc and Tab still work cleanly
    - On close, focus is returned to whatever element opened the modal
      so keyboard users aren't dumped at <body>

POLISH: Modal.confirm Accepts okLabel / okClass

  Modal.confirm() previously always rendered a red btn-danger "Delete"
  button regardless of what the dialog was actually doing. New optional
  fourth arg {okLabel, okClass} lets callers pick. Updated non-
  destructive call sites:
    - SetupWizard.loadDemoData() — "Load demo" / btn-primary
    - Strains.handleDelete() — "Archive" / btn-primary
    - CloudSync.restore() — moved entirely to the diff preview path
  Existing destructive call sites (clone batch delete, rotation delete,
  pheno hunt delete, etc.) are unchanged.

PERF: Lazy-Loaded Photos in List Views

  Eight list/grid img render sites now carry
  loading="lazy" decoding="async":
    - Plants tab plant cards (m.photo)
    - Journal tab grid (.journal-img)
    - Harvest tab harvest cards
    - Strains tab strain cards
    - Strain Reviews photos (.review-photo)
    - Plant Timeline thumbs (.tl-thumb)
    - Grow Journal Export photo grid
    - Photo Compare thumb grid (.ptc-thumb)
  Form previews (single visible image at a time) are not lazied. Net
  effect: a journal with 200 entries no longer pays the full memory
  cost on render — only the photos that scroll into view decode.


v2.22.0
--------------------
FEATURE: Demo Data Lifecycle — _demo Flag + Clear Demo Data

  Every record the SetupWizard demo fixture creates is now tagged with
  _demo:true at creation time. The fixture uses a snapshot-before-build
  pattern: keys of every collection are captured before the fixture runs,
  then anything not in the snapshot is tagged _demo:true at the end. This
  catches every record type without per-record edits, and is safe against
  future additions (any new record a future fixture writes is auto-tagged
  if it lands in a recognized collection).

  New DemoData module:
    - DemoData.hasDemo()  — boolean, true if any _demo record exists
    - DemoData.countDemo() — per-category breakdown
                              { mothers:6, harvests:5, feedings:40, ... }
    - DemoData.clearDemoData() — shows a confirm modal with the per-
                                 category breakdown, then surgically
                                 removes only _demo records (user records
                                 untouched)
    - DemoData.isBannerDismissed() / dismissBanner()
                              — banner-state helpers

FEATURE: Dashboard Demo Banner

  When demo data is detected and the user hasn't dismissed the banner, a
  persistent banner appears at the top of the dashboard (between hero and
  stats grid) with three actions:
    - Clear demo data  — invokes the confirm modal + surgical removal
    - + Add my plant   — opens Plants.openAddModal() directly
    - Keep mixed       — sets demoBannerDismissed=true, hides the banner
                         until demo data is cleared or re-loaded
  The "Keep mixed" choice is reversible via the Command Palette entry
  "Show Demo Banner" which clears the dismiss flag.

FEATURE: Three Demo-Related Command Palette Entries

  Ctrl/Cmd-K → "Clear Demo Data" surfaces the same confirm flow as the
  banner button. "Show Demo Banner" restores the banner if it was
  dismissed via "Keep mixed". "Load Demo Data" remains available for
  loading the fixture (with the existing confirm-before-clobbering
  prompt if real data is present).

  Also added to the gear menu under Quick Actions as
  "🧹 Clear Demo Data" for users who never see the banner.

v2.21.3
--------------------
FEATURE: Sharable Demo URL

  Visiting the app with "?demo" in the URL (or "#demo" hash) now auto-
  loads the demo fixture on init. Useful for sharing a preview link

v2.21.2
--------------------
FIX: HarvestEstimate._parseFloweringTime crashed on numeric input

  The function assumed floweringTime was always a string ("8 weeks" or
  "55-65 days") and called str.toLowerCase() unconditionally. Strain
  catalog entries and the demo-data fixture both store floweringTime as
  a Number, which caused "str.toLowerCase is not a function" the moment
  any flowering plant existed and the dashboard tried to render its
  HarvestForecast card. Loading demo data manifested it because the
  fixture creates two flowering plants on first call.

  _parseFloweringTime now (1) accepts numbers directly and treats them
  as days, (2) coerces unknown types to string before regex matching,
  and (3) handles bare-number strings ("65"). Also hardened
  _findFloweringTime and HarvestForecast._strainYieldStats to coerce
  m.strainName before .toLowerCase() in case a record is missing it.


v2.21.1
--------------------
FIX: Service Worker Was Serving Stale index.html

  The service worker had three issues that combined to lock users on a
  stale shell forever: (1) CACHE_VERSION was stuck at 2.5.0, so old caches
  never got purged on activate; (2) PRECACHE referenced /PlantManager.html
  which doesn't exist (left over from a rename — the actual file is
  /index.html); (3) the same-origin fetch handler was cache-first, so once
  a stale index.html got cached, the user kept getting it forever and
  never saw new deploys. This is why "Load Demo Data" appeared broken on
  the live site — users were running the old, sparser fixture.

  Switched HTML/text shell requests to network-first with cache fallback,
  removed index.html from PRECACHE entirely, bumped CACHE_VERSION to
  2.21.1 (which purges every old trm-pm-* cache via the activate handler),
  and added a SKIP_WAITING + controllerchange-reload flow on the page so
  fresh deploys take over on the next load instead of waiting for all
  tabs to close.

  Users currently on a stale shell may need ONE hard reload
  (Ctrl/Cmd+Shift+R) to break out the first time. After that, future
  deploys land automatically on the next normal reload.
